# Flywheel Playbook — Data Directory

This directory stores all portfolio data, trade history, and analytical outputs. Subagents read from and write to these files.

## Files

Updated manually or by the portfolio-accountant when trades are recorded.

        "underlying": "NVDA",
        "direction": "SHORT",
        "type": "CALL",
        "strike": 205,

### `trades.json` — Trade History Ledger
Append-only log of all transactions. Used for income metrics, cost basis calculation, and performance tracking.

```json
[
  {
    "id": "m1abc123",
    "date": "2026-03-17",
    "account": "ROTH",
    "action": "STO",
    "ticker": "NVDA",
    "type": "CC",
    "strike": 205,
    "expiration": "2026-05-01",
    "premium": 4.40,
    "qty": 4,
    "notes": "Mode 1, 45-DTE, Double Barrier at $205/0.20Δ"
  }
]
```

**Action codes:**
- `STO` — Sell to open (new CC or CSP)
- `BTC` — Buy to close (closing existing position)
- `ROLL` — Roll (BTC + STO in one entry, note net credit)
- `ASSIGNED` — Option was assigned (shares called away or put to you)
- `EXPIRED` — Option expired worthless
- `BUY` — Share purchase (swing trade or direct buy)
- `SELL` — Share sale
- `DIVIDEND` — Distribution received (JEPQ/JEPI)
- `SPAXX` — Money market yield recorded

### Analytical Outputs (auto-generated by subagents)

| File Pattern | Source | Content |
|---|---|---|
| `weekend-session-YYYY-MM-DD.md` | weekend-session | Full weekend analysis with decision tree |
| `roll-eval-{TICKER}-{DATE}.md` | roll-evaluator | Individual position roll analysis |
| `monday-scan-YYYY-MM-DD.md` | monday-scanner | Turnaround Tuesday go/no-go checklist |
| `macro-force-YYYY-MM-DD.md` | macro-analyst | Weekly force assignment map |

## Getting Started

1. Copy the `positions.json` template above and fill in your actual holdings
2. Start `trades.json` as an empty array `[]` and begin logging trades
3. Run `/status` to verify the portfolio-accountant reads your data correctly
4. Run `/weekend` on Saturday or Sunday to generate your first session report
